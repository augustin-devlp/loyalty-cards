BUG #2 - Routes API Sans Try-Catch Global [HAUTE]
────────────────────────────────────────────────────
Fichiers:   src/app/api/admin/generate-code/route.ts
            src/app/api/admin/activate/route.ts
Problème:   Exceptions non capturées → réponses HTTP non-JSON possibles
Fix:        Enveloppe try-catch global autour de POST()

BUG #3 - Parsing du Body JSON Non Protégé [MOYENNE]
─────────────────────────────────────────────────────
Fichiers:   src/app/api/admin/generate-code/route.ts
            src/app/api/admin/activate/route.ts
Problème:   `await req.json()` peut lever exception → pas de catch séparé
Fix:        Try-catch séparé avec retour NextResponse.json({ error: "Body invalide" })

BUG #4 - Gestion JSON Côté Client Incohérente [MOYENNE]
─────────────────────────────────────────────────────────
Fichier:    src/components/AdminPendingActions.tsx
Problème:   `await res.json()` peut échouer si réponse non-JSON
Fix:        Fonction `safeJson()` qui parse texte puis JSON avec fallback

BUG #5 - Vérification du Code Manquante [MOYENNE]
──────────────────────────────────────────────────
Fichier:    src/components/AdminPendingActions.tsx
Problème:   N'affiche aucun code si API retourne `{ ok: true }` sans `code`
Fix:        Ajout de `if (!data.code) throw new Error("Aucun code retourné")`
